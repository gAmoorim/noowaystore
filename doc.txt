Documentação de Rotas e Controllers

🔹 Usuarios

controllerCadastrarUsuario (POST /usuarios)
# - Body: nome, email, senha, telefone (opcional)
# - Validar campos obrigatórios
# - Validar formato de email
# - Senha com pelo menos 6 chars
# - Verificar se email já existe
# - Criptografar senha com bcrypt
# - Retornar 201 + (id, nome, email, tipo)

controllerLogin (POST /login)
# - Body: email, senha
# - Validar campos obrigatórios
# - Verificar se email existe
# - Comparar senha com bcrypt
# - Gerar token JWT com: id, nome, tipo
# - Retornar 200 + token + dados do usuário

controllerListarUsuarios (GET /usuarios) (Restrito a administradores)
# - Verificar se usuário logado é admin
# - Buscar todos os usuários (exceto senha_hash)
# - Retornar 200 + lista
# - Erros: 403 (não admin)

controllerObterUsuario (GET /usuarios/:id)
    # - receber id do usuario no params
    # - Requer autenticação
    # - Usuário acessa apenas o próprio perfil
    # - Administrador acessa qualquer usuário
    # - Busca usuario por ID
    # - retorna: 200 OK + dados do usuário

controllerAtualizarUsuario (PUT /usuarios)
    # Body: nome, email, senha, telefone (opcional)
    # Requer autenticação
    # se senha enviada: recriptografar
    # se email enviado: verificar se já existe
    # Usuário atualiza apenas o próprio
    # atualiza dados
    # retorna: 204 No Content ou 200 OK

controllerDeletarUsuario (DELETE /usuarios/:id)
    # Requer autenticação
    # Remove usuário
    # Retorna: 204 No Content

🔹 PRODUTOS

controllerCriarProduto (POST /produtos)
# - Verificar se usuário logado é admin
# - Body: nome, descricao, preco, categoria_id, disponivel (default true)
# - Validar campos obrigatórios (nome, preco)
# - Validar categoria existente (se informado)
# - Criar produto
# - Retornar 201 + produto

controllerListarProdutos (GET /produtos)
# - Listar produtos (com categoria, imagens, estoque)
# - Suporta filtros: nome, preco_min, preco_max, categoria_id, disponivel
# - Retornar 200

controllerBuscarProduto (GET /produtos/:id)
# - Buscar produto pelo id
# - Incluir imagens, estoque, categorias
# - Retornar 200

controllerAtualizarProduto (PUT /produtos/:id)
# - Verificar se usuário logado é admin
# - Validar dados enviados ( nome, descricao, preco, disponivel)
# - Verificar se produto existe
# - Atualizar campos enviados
# - Retornar 200 + produto atualizado

controllerDeletarProduto (DELETE /produtos/:id)
# - Verificar se usuário logado é admin
# - Verificar se produto existe

🔹  PEDIDOS

controllerCriarPedido (POST /pedidos)
# - Verificar autenticacao
# - Body: itens = [{ estoque_id, quantidade }] (array)
# - verificar se cada item existe no estoque
# - Verificar se quantidade solicitada ≤ disponível
# - Criar pedido com status = 'pendente'
# - Criar itens_pedido com preco_unitario = preco do produto
# - Atualizar estoque (quantidade -= quantidade)
# - Retornar 201 + pedido completo
